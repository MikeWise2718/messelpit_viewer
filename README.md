# Messel Pit Viewer

NVIDIA Omniverse Kit application for visualizing the **Grube Messel** UNESCO
World Heritage fossil site (near Darmstadt, Hesse) as a textured LiDAR
heightfield, with a domain extension for camera viewpoints and an info panel.
Built to also stream to a **Meta Quest 2 / Quest 3** browser via the
CloudXR.js / WebXR pipeline.

This repo is the **viewer**. The **data pipeline** that produces `messel.usd`
lives in a sibling repo at `D:\senckenberg\messelpit\`.

## Layout

```
messelpit_viewer/
  kit-app-template/                          (vendored — see VENDORED_FROM)
    source/apps/
      senckenberg.messelpit.explorer.kit     desktop iteration
      senckenberg.messelpit.viewer.kit       Quest streaming target
    source/extensions/
      senckenberg.messelpit/                 domain extension (auto-load,
                                             viewpoints, info panel) —
                                             shared by both kit apps
      senckenberg.messelpit.explorer.setup/  template setup, untouched
      senckenberg.messelpit.viewer.setup/    template setup, untouched
      senckenberg.messelpit.viewer.messaging/ Quest messaging
    .vscode/replay_files/                    non-interactive scaffolding
  specs/                                     design notes
  launch.bat                                 one-line desktop launch
  README.md                                  this file
  VENDORED_FROM                              upstream commit of kit-app-template
  .gitignore
```

`senckenberg.messelpit/` is where all our domain logic lives. The two
`.setup` extensions and the `.messaging` extension are stock kit-app-template
scaffolding generated by `repo template replay`.

## Prerequisites

- **Windows 10/11** (Linux supported by upstream, untested here)
- **NVIDIA RTX GPU** (RTX 3070 or better recommended)
- **Driver ≥ 551.78** (Windows)
- **Git** ([download](https://git-scm.com/downloads))
- **Visual Studio 2019 or 2022** with the *Desktop development with C++*
  workload (only required if building C++ extensions; the current scaffold
  is Python-only so it's optional)

## Bootstrap a new machine

```powershell
git clone <this-repo-url> messelpit_viewer
cd messelpit_viewer\kit-app-template
.\repo.bat build
```

First build downloads the Kit SDK (~multi-GB) into the global
`%LOCALAPPDATA%\ov\data\` cache and takes ~5 minutes. Subsequent builds reuse
the cache and are fast.

You also need the **data repo** for `messel.usd`. By default the launch
script looks at `D:\senckenberg\messelpit\out\messel.usd`. Override via the
`MESSEL_USD` env var (see launch.bat).

## Launch

```powershell
.\launch.bat
```

This launches the Explorer kit with auto-loading enabled. The viewer's
domain extension reads `--/app/messelpit/load_usd=<path>` on startup and
opens the stage. To skip auto-load:

```powershell
.\launch.bat --no-auto
```

To point at a different USD:

```powershell
set "MESSEL_USD=D:\path\to\other.usd" && .\launch.bat
```

First launch takes 5–8 minutes for shader compilation; subsequent launches
are seconds.

## What's where

- **Camera viewpoints**: `senckenberg.messelpit/senckenberg/messelpit/viewpoints.py`.
  Coordinates in local meters; SW corner of the bbox is (0, 0, 0). Adding a
  viewpoint here makes it appear in the desktop side panel and (once the
  VR UI is wired up) on the Quest.
- **Auto-load path**: read from setting `/app/messelpit/load_usd`, set by
  `launch.bat`.
- **Controls panel**: `senckenberg.messelpit/senckenberg/messelpit/ui_desktop.py` —
  docked next to the Stage hierarchy by default.
- **Spec & open decisions**: `specs/messelpit-viewer.md`.

## Data pipeline (separate repo)

The data repo at `D:\senckenberg\messelpit\` produces `messel.usd`,
`messel.usdz`, and `ortho.png`. **Important**: the orthophoto is capped at
16384 px on the long axis to satisfy the D3D12 `Texture2D` limit — without
the cap, the texture fails to upload in Kit (although it works in `usdview`
which uses Storm/OpenGL). See `tools/prep_rasters.py --max-tex-dim` in the
data repo.
